# Assessment Form IA and Wireframes

## Information Architecture

- Entry state: page shell, compact intro panel, level selector, not-now/start actions.
- Assessment state: title/context, progress, domain, current question, answer choices, next/submit action.
- Result state: existing result flow remains in place.

## Desktop Wireframe

- Header row: assessment name, level context, generated/curated source.
- Left rail: question count, percent complete, level, domain, time expectation.
- Main panel: domain badge, question, answer list, inline next/submit action.
- Error state: stays above the cockpit and does not move controls.

## Mobile Wireframe

- Compact page heading below navbar.
- Progress strip with question count and percentage.
- Question panel starts earlier in the viewport.
- Answer cards use tighter padding and stable tap targets.
- Sticky bottom action bar keeps next/submit reachable.

## Content Strategy

- Use direct copy: "Choose your starting level", "15 questions", "3-5 minutes".
- Keep level descriptions short and plain.
- Do not add unsupported outcomes or claims.
